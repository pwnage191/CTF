#### Challenge Name
Directory Traversal via Gallery Parameter

#### Vulnerability Type
Path Traversal + Improper Access Control + Sensitive File Exposure

#### Goal
To access the hidden `password.txt` file and retrieve the password

#### Initial Observation
- Found a parameter: `?galerie=`
- Noticed it was used to load images in the gallery
- The value of the parameter was directly reflected in the file path
- Suspected lack of input validation

#### Exploitation
- `../../` triggered `open_basedir` error, confirming traversal was limited to one level
- `opendir()` errors confirmed the server lists directories, not reads files
- Used `galerie=./` to explore the current directory
- Discovered hidden folder: `86hwnX2r`
- Gallery listed all files inside the folder
- Found `password.txt`
- File was rendered using `<img src="...">`
- Opened image in new tab → accessed raw file
- Password was revealed

#### Remediation
- Validate and sanitize user input (block `../`, `./`)
- Use whitelist for allowed directories/files
- Never expose PHP error messages to users, they leak server structure and logic

#### What I learned
- User input should never directly control file paths
- Directory traversal can expose hidden/sensitive files
- Frontend rendering (like `<img>`) can bypass backend logic
