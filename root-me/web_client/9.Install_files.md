#### Challenge Name  
- phpBB – Install Files Exposure    
  
#### Vulnerability Type  
- Misconfiguration    
- Exposure of Installation Files    
  
#### Goal  
- Identify and access leftover installation files to gain unintended functionality    
  
#### Initial Observation  
- The challenge hint mentioned **install files** and phpbb.
- I was not sure what that meant initially    
  
#### How I approached it (Methodology)  
- I searched:  
  - "phpBB install files"  
  - "phpBB directory structure"  
  
- Learned:  
  - phpBB uses an `/install/` directory during setup    
  - This directory should be deleted after installation    
  - If not removed, it can still be accessed    
  - This gave me the idea to check for leftover install paths    
  
#### Recon  
- Tried common paths:  
```
/install/  
/install/index.php  
/install/app.php
```
- Found that `/install/` was accessible

#### Exploitation
- Accessed the `/install/` endpoint
- The flag was found directly inside the install page

#### Remediation
- Remove the `/install/` directory after installation
- Restrict access to setup-related files

#### What I learned
- Always understand the technology using simple searches
- Check for common leftover paths like `/install/`
- Small hints can directly point to vulnerabilities
